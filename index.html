「Webコンテンツ作成」の成果

本テーマの学習目的

情報工学の専門科目の基礎知識を応用して実践的な能力を修得することが目的です。到達目標は、 一つ目として、手順書を読み、実験を行い、レポートやプレゼンテーションにより報告するという実験の一連の流れを実践できること。 二つ目として、HTML等を用いたWebページ作成ができること。Webコンテンツ作成をするにあたって、この二つが重要になります。

プロジェクト課題について

プロジェクトの課題では、世界の天気を表示するウェブページを開発しました。ユーザーは都市名のIDを入力して検索することで、 都市名、最高気温、最低気温、湿度、緯度、経度、天気、風速、風向が即座に表示されます。 これにより、ユーザーは手軽に興味のある都市の天気情報を取得できます。さらに、利便性を高めるために各都市のIDをページに掲載しました。 開発過程では、外部APIとの連携と非同期処理の理解が重要であることがわかりました。デザイン面では、ページの見た目や使いやすさを重視し、シンプルでなデザインにしました。 また、新たな機能の追加や地域の天気情報の表示など、さらなる拡張もできるなと感じました。 このプロジェクトを通じて、外部APIとの連携や非同期処理などのプログラミングスキルが向上し、私にとって価値あるプロジェクト作成になりました。

数当てゲーム

この課題では、まず、ユーザは1〜10の値を1つ決めて、答えを予想する。 予想は3回できて、ユーザの入力が外れた場合は、入力よりも答えが大きいか小さいか表示する。 3回以内に答えの数を当てればユーザの勝ち、そうでなければ負けというのがこのゲームの仕様である。この課題で苦労した点が何個かあった。まず、ユーザーが1〜10の範囲外の数値を入力した場合の処理を行うのにてこずった。 次に、正解を入力しても、正解と表示されないということが起こった。この表示方法の違いについて、if文の構成が複雑で難しかった。他にも、ユーザーに3回までの予想をさせるために、 ループと回数制限を組み込む必要があり、ループの動作や制御を理解するのが難しかった。この課題を通じて、プログラムのデバッグやテストの重要性も理解することができた。 プログラムが期待通りに動作するかどうかを確認するために、定期的に入力のテストや変数の値を確認することが大切であると感じた。

第4回 DOM と JavaScript

今回は、DOMについて詳しく学んだ。DOMの役割として、JavaScriptなどのプログラムにより、Webコンテンツの内容を読み取ったり、 コンテンツに変更を加えたりする処理が書けるようになることが分か理ました。実際に使ってみると、やり方を覚えてしまえば、いろいろなことができるのだと理解しました。 画像を貼ったり、箇条書きを追加、削除したりなど、今回でできることがとても増えました。

第5回 イベント処理

今回は、イベントについて理解を深めることができました。 イベントとは、Webページ上でユーザーが行う動作や発生する出来事のことを指します。例えば、ボタンのクリックやマウスの動きなどがイベントになります。 イベント処理とは、JavaScriptを使ってイベントが発生した際に動作する処理を実現する方法のことで、 この仕組みを学ぶことで、ボタンを押すとメッセージが表示されたり、画像が変わるようなプログラムを作成することができるようになりました。

工夫点，苦心点，強調点，感想

全ての講義を通して、感じたことは、TML、CSS、JavaScriptを使用してWebページを作成する経験は初めてだったので、 プロジェクト課題ではいくつかの苦労点や興味深い発見がありました。私は世界の天気を表示するウェブページを作成しましたが、 最初はAPI連携と非同期処理に戸惑いました。しかし、理解を深めることで天気情報を表示する機能を実現することができました。 また、数当てゲームではJavaScriptを使ってユーザーとの対話的な体験を追求しました。DOM操作や条件分岐などの基本的なプログラム概念を学び、 特に'addEventListener'を使ってボタンがクリックされた時の処理を設定する方法に興味を持ちました。プレイヤーが予想を入力し、 結果が表示されるところがこのゲームの魅力であり、DOM操作を通じてHTML要素の内容を変更し、プレイヤーにフィードバックを提供することで、 ゲームの進行状況をわかりやすく伝えることができました。ただし、今回作成したゲームは簡単なものであり、より多様な機能を追加することで、 楽しさや興味を引き出す可能性を感じました。これからはゲームの改善と新機能の追加に挑戦したいと思っています。 デザイン面では、全体的な色やレイアウトを鮮やかな色にし、見やすく使いやすい、見ていて楽しいウェブページを目指しました。 苦労した点について、特にDOM操作に関しては、要素の検索やテキストの変更、新しい要素の追加などで正確なコードを記述し配置することに苦労しました。 数当てゲームでも表示設定や条件分岐に手間取ったり、if文の作成にあたり、javaのプログラムと書き方が少し異なるのと、 エラーにはならずに求めたい結果が表示されなかったので、エラーの場所がわからず戸惑いました。しかし、先生の説明を受けることで理解が進みました。 全体的な感想としては、HTML、CSS、JavaScriptを使ってWebページを作成する経験は初めてでありながら、多くの知識とスキルを身につけ、 プログラミングの面白さと奥深さを実感することができました。最初は難しく感じる部分もありましたが、課題に取り組むうちに理解が深まっていく過程が楽しいと感じました。 自分の考えを実現するためにコードを書くことは創造的でやりがいがあります。今後はより高度なWebページやアプリケーションを作成するために知識を深め、 新しい技術や開発手法にも積極的に取り組んでいきたいと思うことができました。これからも精進してより素晴らしいウェブページを作成できるよう努力していきたいと思います
